Deploy and hydrate the local two-chain stack

Forward path

Lock on Bank A, then prove on Bank B

Commit the packet, finalize the header, update the remote client, and execute the storage proof before the voucher can exist on Bank B.

Lending use case

Use the proven voucher as collateral

The lending pool is downstream from the proof. It demonstrates what verified cross-chain state can unlock, not what the system trusts.

Reverse path

Burn the voucher and unescrow on Bank A

The return trip mirrors the forward path: commit on Bank B, trust the new header on Bank A, and then unlock the escrowed asset.

Safety path

Freeze on conflicting updates, recover explicitly

Misbehaviour remains first-class: submit a conflict, freeze the client, then recover under a successor epoch to restore trust progression.
